# Excellusion

A spreadsheet with no spreadsheet. **[Try it live](https://lufzle.github.io/excellusion/)**

> *The simulacrum is never that which conceals the truth — it is the truth which conceals that there is none.*
> — Jean Baudrillard

## The argument

Software is a sequence of states transitioning according to patterns. Those patterns are learnable. Which means they don't need an explicit, hand-written implementation to look computed — they just need to be predicted well enough to satisfy the contract.

When you type `=A1+1` and see `11`, was the answer computed or predicted? If the contract is honored — same results, same invariants, same edge cases — what exactly is the meaningful difference?

Key points from the accompanying blog post:

- World models (Genie 3, LingBot-World) generate interactive environments with no game engine, no 3D models, no code — just `(history + action) → next frame`.
- The same principle applies to any software: a spreadsheet, a terminal, a file system. They are all learnable behaviors.
- We don't verify software by inspecting its mechanism. We treat it as a black box: feed input, assert output. If the tests pass, the contract is honored.
- Two boxes — one running a formula parser, one running a model that predicts what a formula parser would output — are indistinguishable if both satisfy the same contract.

[Read the full essay: *Nobody Wrote the Matrix*](https://lufzle.fika.bar/nobody-wrote-the-matrix-01KJNMM4B53CB0DYHXFMQ0FS1A)

## What this project demonstrates

Excellusion is a working spreadsheet powered entirely by an LLM (Claude Haiku 4.5), with **zero business logic in the code**. It comes in two modes that represent two levels of the same idea:

### Hybrid mode

The UI exists as React code — the grid, the formula bar, the cell selection. But there is **no data layer**. No variables hold cell values. No formula parser evaluates expressions. When you edit a cell, the raw edit is sent to the LLM, which returns the full grid state as JSON. The React code simply renders whatever comes back.

The building has a front door, but nothing behind it.

### Full mode

Even the UI is gone. The **entire spreadsheet** — every pixel, every cell border, every formula bar character — is generated by the LLM as pure static HTML. When you click, the browser takes a screenshot (with a red crosshair marking the click position), serializes the raw DOM event, and sends both to the LLM. The LLM looks at the image, interprets your action, and returns the next frame.

There is no building at all.

## How to use it

### Prerequisites

- Node.js 22+ (managed via [mise](https://mise.jdx.dev/))
- An [Anthropic API key](https://console.anthropic.com/)

### Setup

```bash
pnpm install
pnpm dev
```

Open http://localhost:3003 in your browser.

Paste your Anthropic API key in the top bar. The key is stored in `sessionStorage` (cleared when you close the tab) and never sent to any server — all LLM calls go directly from your browser to the Anthropic API.

### Usage

- **Hybrid tab**: Click a cell, type a value or formula (`=A1+B1`, `=SUM(A1:A5)`), press Enter. The LLM computes the result.
- **Full tab**: Click or type anywhere. Watch the LLM regenerate the entire spreadsheet UI from a screenshot of what it looks like and the raw browser event you triggered.
- **Log pane**: Shows every request/response. Click "System prompt" to see the exact prompt and model configuration.

## Technical details

- **Model**: Claude Haiku 4.5 (`claude-haiku-4-5-20251001`)
- **Prompt caching**: System prompt and conversation history are cached using Anthropic's ephemeral cache to reduce latency and cost.
- **No server**: All API calls are made directly from the browser using `@anthropic-ai/sdk` with `dangerouslyAllowBrowser: true`.
- **Screenshot capture**: Full mode uses `html2canvas` to capture the iframe content and draw a click indicator before sending to the LLM as a base64 image.
- **Framework**: React 19 + Vite + Tailwind CSS. Pure SPA — no server, deployable to GitHub Pages.
